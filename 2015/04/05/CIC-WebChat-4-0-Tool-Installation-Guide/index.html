CIC WebChat 4.0 Tool Installation Guide

Call Centre
  • CIC
  • WebChat

Purpose

This guide is a goal for CIC WebChat implementation evaluation.

Prerequisites

  • A web server with Windows 2008 R2 & II7.5
  • A IC server 4.0 (Only one for this scenario,Not HA)
  • A client PC with Interaction Client:.Net Edition installed
  • WebSvcsConfiguration.zip & WebSvcsI3Root.zip, these two zip files locate in //ICServer/IC_WebServicesChat, ICServer is the server installed IC 4.0 package.

In this scenario, we only focus on Windows & IIS platform. For other web server installation,please refer to the original document.
And also, we do not cover IC server 4.0 installation,we assume that IC server and the appropriate licenses are all done for this scenario.

Installation & Configuration Steps

IC Server

  • Create a Chat ACD queue named “TestChat” by IA tool.

  • Enable queue ACD， Active，Record All Calls， Emails，Chats and Instant Questions in this workgroup functionality.

    .

    Select “ACD” Tab on “Workgroup Configuration - TestChat”,

    Set the value of Chat Max.Assign >=1, this value can be overwritten later for specific agent based on his capability.

  • Create an agent account named “TestAgent”.
  • Create Station with Chat & ACD access license.

    Web Server

  • Download ARR and install it on web server. Using this to rewrite the chat request sent by the customer to the back-end IC server.
  • Create /I3Root/Server1 directory under IIS default web site , this we use the default IIS root directory C:/inetpub/wwwroot

  • Enable https, select hostname via IIS Manager console and double click Server Certificates on right pane.See the blow figure.

    For this evaluation purpose, we use “Create Self-Signed Certificate”.

    After the above steps, Click “Binding…” from Actions pane after selecting “Default Web Site” on IIS Manager to add https port binding.

  • Setup URL rewrite Rule, Select “Server1” item under “Default Web Site” -> “I3Root”, double click “URL Rewrite”,

    click “Add Rule(s)…” from the right Action pane then add rule window will pop up as below:

Select “Reverse Proxy” and click “OK” button,the following window will show,

type ICServer in the Enter the server name or the IP address where HTTP requests will be
forwarded text box, clear “Enable SSL offloading” causing the connection between the customer and the web server, the connection between the web server and the IC server are both SSL.
We strong recommend, put the Full IC Server’s hostname here and make sure that the web server can refer to the IC Server using the IC Server hostname via DNS or the web server’s host file.After finish rules addition,we will get the result as.

Double click this rule,scroll down and locate the Action section in the pop window,

key in such information {C:1}://ICServer Full hostname:3508/{R:1} in “Rewrute URL” text box, then click “Apply” button on the right “Action” pane.

  • Unzip WebSvcsI3Root.zip, retrieve contents under I3Root directory and put them into web site, here is C:/inetpub/wwwroot/I3Root.On other hand, if not create I3Root under C:/inetpub/wwwroot previously, this can copy or move the unzip contents into C:/inetpub/wwwroot including I3Root directory .

  • Run the interaction Web Tools Configuration Utility,

    this tool is included into WebSvcsConfiguration.zip. If you only want to have Chat, Clear “call back” within Interaction Type section. Because we using SSL among Web server and IC Server, please make “Secure Connetion(HTTPs)” enable.

  • Copy ServerGroupCertificate.cer to the web server, make sure this file must not be accessed from web site. This CA file is located D:/I3/IC/Certificates/ServerGroup on IC Server.Note:If IC Server name is change,the file must be regenerated,and we re-import certificate by the following step. 1.Opne MMC Console via key the mmc command in OS’s run box.Click “Add/Remove Snap-in…”,

    select “Certificates” then click “Add” button,choice “Computer account” then press “Next”,select “Local computer” and click “Finish” after that press “OK”.Expand “Trusted Root Certification Authorities” under “Certificates” node, selected “Certificates” node under “Trusted Root Certification Authorities”.Right-click this “Certificates” node and select “All Tasks” -> “Import”.

    After that ,find and import the ServerGroupCertificate.cer.

Validation

  • Using https://icservername:3508/websvcs/serverConfiguration to check IC Server work as expected.
  • Using https://webserver/I3Root/Server1/websvcs/serverConfiguration to check URL Rewrite as expected.
  • Using https://webserver/I3Root/ and see the following pictures,

it shows The WebChat works fine.

Note:For Chat real time monitoring is same as call’s, we can retrieve the real time statistics via IC Business Manager.

Now, we can say our first CIC WebChat scenario is done!
